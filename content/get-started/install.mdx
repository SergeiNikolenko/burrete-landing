# Install Burrete

Burrete supports macOS 12 and later.

## Homebrew

Install from the Burrete tap:

```bash
brew tap SergeiNikolenko/burrete
brew install --cask burrete
```

Check the installed cask version:

```bash
brew info --cask SergeiNikolenko/burrete/burrete
```

## Bun CLI

Use the CLI when you want the latest stable GitHub release or a quick health check:

```bash
bunx burrete latest
bunx burrete install
bunx burrete doctor
```

The Bun installer places Burrete in `~/Applications` by default. Use `bunx burrete install --system` only when you intentionally want the app in `/Applications` for all users.

## GitHub Releases

You can also install manually:

1. Open the [latest release](https://github.com/SergeiNikolenko/Burrete/releases/latest).
2. Download the `Burrete-<version>.zip` file.
3. Unzip it.
4. Move `Burrete.app` to your `Applications` folder.
5. Open Burrete once from `Applications`.

## First launch

Open `Burrete.app` once after installation. This gives macOS a chance to register the app bundle and Finder Quick Look extension.

If Finder previews do not appear after install, run:

```bash
bunx burrete doctor
```

## Codex plugin and local MCP

The optional Codex integration lives in the same [Burrete repository](https://github.com/SergeiNikolenko/Burrete) as the app. It bundles workflow skills with typed MCP tools for opening, observing, and controlling a local molecular workspace.

Clone the repository and run the plugin installer:

```bash
git clone https://github.com/SergeiNikolenko/Burrete.git
cd Burrete
bun run install:plugin
```

Restart Codex after installation, then mention `@Burrete` or select Burrete from Plugins. See [Codex Plugin and MCP](/docs/plugin) for the tool surface, security boundaries, and source links.

## Optional local calculation engines

The Burrete application does not bundle xTB, CREST, or PRISM. Install only the engines needed for your local workflow:

```bash
pixi global install xtb
pixi global install crest
uv tool install prism_pruner
```

Check each executable before a production calculation, then use the matching runtime check in the desktop inspector. See [xTB calculations](/docs/workflows/xtb) and [CREST and PRISM](/docs/workflows/crest-prism).
