# Desktop Workspace

The desktop app is the full Burette workspace. Use it when a one-off Finder preview is not enough.

![Burette desktop workspace with project navigation, tabs, a Molstar structure view, and inspector docks.](/assets/main-light.png)

*The desktop surface keeps the active structure, project navigation, and workflow panels in one window.*

## Core workflow

- Keep previews alive in tabs while you switch files.
- Browse project folders, nested structures, recent files, and search results from the sidebar.
- Use the command palette for opening files, switching renderers, maintenance actions, exports, project navigation, Ketcher, and FEP network previews.
- Compare structures, text files, Ketcher, grids, and workflow panels in right and bottom docks.
- Open files in Finder, the default app, or discovered chemistry editors.
- Drag files, tabs, grid rows, sketches, or recognized structure text into target-aware workspace actions.

## Collection work

Burette includes an RDKit-powered collection grid for SDF, SMILES, CSV, and TSV files. The grid supports search, sorting, SMARTS filtering and highlighting, selection, incremental loading, append and merge workflows, editing, and export.

Read [SDF and molecule collections](/docs/workflows/collections) for exact merge rules and [Drag and drop](/docs/workflows/drag-and-drop) for target behavior.

## Ketcher scope

The desktop app includes a Ketcher small-molecule editor. You can sketch a molecule, import and export common small-molecule formats, send a sketch to Mol*, `xyzrender`, or a collection grid, and edit grid rows through Ketcher.

Macromolecule editing is not enabled in the current Ketcher integration.

## Local calculations

The structure inspector exposes local xTB calculations and CREST/PRISM conformer workflows. They require separately installed executables, keep job status and artifacts visible, and reject direct chemistry jobs above the 300-atom guard unless you select a smaller molecular object.

- [xTB calculations](/docs/workflows/xtb)
- [CREST and PRISM](/docs/workflows/crest-prism)
